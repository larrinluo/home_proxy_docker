## 一、整体布局设计

### 1.1 页面整体结构

页面采用经典的三栏布局结构，包含顶部导航栏、左侧边栏和主内容区。

```
┌─────────────────────────────────────────────────────────┐
│  顶部导航栏 (高度: 60px)                                  │
│  家庭网络 | GitHub | Support | Discord                   │
├──────────┬──────────────────────────────────────────────┤
│          │  主标题: 基于SSH跳板服务的Socks代理            │
│          │  代理服务地址: http://192.168.1.4:8090/proxy.pac │
│          ├──────────────────────────────────────────────┤
│ 左侧边栏  │                                              │
│ (150px)  │  ┌──────────────┐  ┌────────────────────┐  │
│          │  │ 代理配置卡片  │  │  [标签页]           │  │
│ Socks代理 │  │              │  │  PAC配置预览        │  │
│          │  │ - 默认配置    │  │  (代码显示区域)     │  │
│          │  │ - 代理服务    │  │                    │  │
│          │  │ - Host配置   │  │                    │  │
│          │  │ - 状态       │  │                    │  │
│          │  └──────────────┘  └────────────────────┘  │
│          │                                              │
│          │  [+ 添加新配置]                              │
│          │                                              │
│ 其他模块  │                                              │
│ - 占位   │                                              │
│          │                                              │
│ 用户信息  │                                              │
│ 系统配置  │                                              │
│ 注销     │                                              │
└──────────┴──────────────────────────────────────────────┘
```

### 1.2 布局层次关系

#### 1.2.1 布局组件层级

1. **顶层容器**

   - 全屏容器，占据整个视口
   - 使用Element Plus的Container布局组件
2. **顶部导航栏（Header）**

   - 固定高度：60px
   - 全宽显示
   - 包含左侧标题和右侧链接
3. **主体容器（Container）**

   - 使用Element Plus的Container布局
   - 包含左侧边栏（Aside）和主内容区（Main）
4. **左侧边栏（Aside）**

   - 固定宽度：150px
   - 垂直布局
   - 包含菜单项和底部功能区
5. **主内容区（Main）**

   - 自适应剩余宽度
   - 包含标题栏、配置卡片、PAC预览等

#### 1.2.2 区域划分

- **顶部导航栏区域**：y坐标 120px - 180px（高度60px）
- **左侧边栏区域**：x坐标 80px - 230px，y坐标 180px - 640px（宽度150px，高度460px）
- **主内容区区域**：x坐标 240px - 1030px，y坐标 180px - 640px（宽度790px，高度460px）

### 1.3 响应式设计断点

#### 1.3.1 断点设置

- **桌面端**：≥1200px

  - 完整布局，所有区域正常显示
  - 左侧边栏固定宽度150px
  - 代理配置卡片和PAC预览并排显示
- **平板端**：768px - 1199px

  - 左侧边栏可折叠
  - 代理配置卡片和PAC预览可垂直堆叠
  - 标签页支持横向滚动
- **移动端**：<768px

  - 左侧边栏改为抽屉式（Drawer）
  - 代理配置卡片全宽显示
  - PAC预览区全宽显示
  - 弹窗全屏显示

#### 1.3.2 响应式适配策略

1. **布局方式切换**

   - 桌面端：Flex布局，横向排列
   - 平板端：Flex布局，支持换行
   - 移动端：垂直堆叠布局
2. **组件适配**

   - 表格：桌面端完整显示，移动端改为卡片式
   - 对话框：桌面端居中显示，移动端全屏显示
   - 按钮组：桌面端横向排列，移动端垂直排列
3. **字体大小适配**

   - 桌面端：保持设计尺寸
   - 平板端：适当缩小（-2px）
   - 移动端：进一步缩小（-4px）

## 五、标签页（Tab）设计

### 5.1 标签页结构和样式

#### 5.1.1 标签页位置

- **位置**：主内容区右侧
- **Host配置标签页位置**：x: 616px, y: 241px
- **PAC配置预览标签页位置**：x: 530px, y: 241px
- **宽度**：根据内容自适应（Host配置标签约144px，PAC配置预览标签86px）
- **高度**：30px
- **位置关系**：标签页水平排列，共享右侧区域

#### 5.1.2 标签页格式

标签页系统包含多种类型的标签页，格式不同：

**1. Host配置标签页格式**

- 显示格式：`Host配置 - 默认配置    x`
- **前缀**："Host配置 - "（固定）
- **配置名称**："默认配置"（动态，根据实际配置名称显示）
- **关闭按钮**："x"（灰色 #b3b3b3，位于标签右侧）
- **特性**：可关闭，根据用户操作动态创建和删除
- **用途**：编辑特定Host配置的Host列表

**2. Host配置标签格式**

- 显示格式：`Host配置`（可能带关闭按钮）
- **标签文本**："Host配置"（固定）
- **关闭按钮**：根据应用需求决定是否显示
- **特性**：应用级标签页，用于Host配置相关功能管理
- **用途**：应用相关的Host配置管理功能

**3. PAC配置预览标签页格式**

- 显示格式：`PAC配置预览`（无关闭按钮）
- **标签文本**："PAC配置预览"（固定）
- **关闭按钮**：无（预置标签，不可关闭）
- **特性**：预置标签，始终存在，不可关闭
- **用途**：显示和预览PAC配置文件代码

#### 5.1.3 标签页样式

- **背景**：浅色背景（#f5f5f5）
- **边框**：无边框或细边框
- **字体大小**：13px
- **高度**：30px
- **圆角**：无圆角（平面风格）

#### 5.1.4 标签页状态

- **激活状态**：背景色稍深，文字加粗
- **非激活状态**：背景色较浅
- **悬停效果**：背景色变化，鼠标指针变为手型
- **PAC配置预览标签页**：始终可用，即使没有Host配置标签页时也显示

### 5.2 标签页交互逻辑

#### 5.2.1 打开标签页的触发方式

**方式1：点击代理配置卡片中的Host配置区域**

- 位置：代理配置卡片中的"Host配置"行
- 示例：点击 "google.com, youtube.com..." 区域
- 行为：打开对应的"Host配置 - {配置名称}"标签页
- 如果标签已存在，则激活该标签

**方式2：通过其他操作触发**

- 创建新Host配置时自动打开标签
- 编辑现有Host配置时打开对应标签

#### 5.2.2 标签页内容区域

当标签页打开后，在标签下方显示内容区域：

```
┌─────────────────────────────────────┐
│ Host配置 - 默认配置            x     │ ← 标签页
├─────────────────────────────────────┤
│                                     │
│  Host列表编辑区域                    │
│  - 显示当前配置的所有Host            │
│  - 支持添加新Host                    │
│  - 支持删除Host                      │
│  - 显示Host冲突提示                  │
│  - 保存按钮                          │
│                                     │
└─────────────────────────────────────┘
```

**尺寸设计**：

**PC端（桌面端）**：
- **高度**：400-500px（足够大，显示完整操作过程）
- **最小高度**：400px（确保显示足够信息）
- **宽度**：与表单同宽（弹窗宽度）
- **弹窗宽度**：建议800-1000px（PC端），确保日志窗口有足够空间
- **支持垂直滚动**：当日志内容超出窗口高度时，支持滚动查看完整日志

**平板端**：
- **高度**：300-400px
- **宽度**：与表单同宽

**移动端**：
- **高度**：250-350px
- **宽度**：全宽显示

**设计原则**：
- PC端日志窗口要足够大，能够同时显示多个操作步骤的日志
- 避免用户频繁滚动，提高操作过程的可读性
- 确保用户能够清楚地看到每个操作步骤的执行情况

## 十一、UI风格规范

### 11.1 Material Design规范

#### 11.1.1 设计原则

- **平面风格**：采用平面设计，不使用阴影效果
- **简洁专业**：参考Google AI Studio的设计风格
- **科技感**：体现专业、科技感的视觉效果

#### 11.1.2 设计元素

- **无阴影**：所有元素不使用阴影效果
- **扁平化**：按钮、卡片等元素采用扁平化设计
- **简洁图标**：使用简洁的图标设计
- **清晰层次**：通过颜色和间距建立视觉层次

### 11.2 颜色规范

#### 11.2.1 主色调

- **主色**：Element Plus默认蓝色（#409EFF）

- **常规文字**：#606266（中灰色）
- **次要文字**：#909399（浅灰色）
- **占位文字**：#B3B3B3（更浅灰色，用于占位功能）

#### 11.2.3 状态颜色

- **成功/运行中**：#67C23A（绿色）
- **信息/已停止**：#909399（灰色）
- **警告**：#E6A23C（橙色）
- **危险/错误**：#F56C6C（红色）

#### 11.2.4 功能颜色

- **链接颜色**：默认链接颜色（可点击状态）

- **英文**：系统默认英文字体
- **代码**：Monaco, 'Courier New', monospace
